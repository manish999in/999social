Social Media API — Routes

Table of endpoints for your backend. Copy/paste URLs into Postman and remember to add Authorization: Bearer <token> for protected routes.

Method Endpoint Auth Description
POST /api/auth/signup No (multipart/form-data for files) Register new user (profilePic & coverPic optional)
POST /api/auth/login No Login (returns JWT token)
GET /api/auth/me Yes Get current logged-in user
PUT /api/auth/me Yes Update current user profile (form-data allowed)
POST /api/posts Yes Create post (form-data: content, description, tags, location, image)
GET /api/posts?page=1&limit=10 No (public) Get paginated feed (newest first)
GET /api/posts/:id No Get single post by ID
GET /api/posts/user/:id No Get posts by a specific user
PUT /api/posts/:id Yes Update post (owner only). form-data allowed to replace image
DELETE /api/posts/:id Yes Delete a post (owner only)
POST /api/posts/:id/like Yes Toggle like/unlike on a post
POST /api/posts/:id/comment Yes Add comment (body: { text })
DELETE /api/posts/:id/comment/:commentId Yes Delete comment (comment owner or post owner)
POST /api/users/:id/follow Yes Follow / Unfollow a user
GET /api/users/:id/followers Yes Get followers of a user
GET /api/users/:id/following Yes Get who the user is following
GET /api/users/ Yes Get all users (for follow suggestions)
GET /api/notifications Yes Fetch notifications for current user
PUT /api/notifications/:id/read Yes Mark notification as read

Tip: For protected routes, set request header Authorization: Bearer <token>. For file uploads use form-data in Postman and set file field name to image (for posts) or profilePic/coverPic (for signup).
